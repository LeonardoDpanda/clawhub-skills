## 1. GitHub 权限

### 需要什么
- **仓库写入权限** (LeonardoDpanda/clawhub-skills)
- **或** 创建 PR 的权限（如果你希望 review）

### 用来做什么
- 自动更新 README.md
- 优化 Skill 文件内容
- 自动提交 A/B 测试版本

### 如何提供
**方案A - 直接权限（推荐，全自动化）**
- 在仓库 Settings > Collaborators 添加我的 GitHub 账号
- 账号: [我的GitHub账号] —— 等等，我需要确认我的GitHub身份

**方案B - PR 模式（你需要 review）**
- 我给你生成修改内容
- 你一键合并 PR

**当前情况**: 我需要先确认我是否有独立的GitHub身份

## 2. Gumroad 权限

### 需要什么
- **Gumroad 账号邮箱 + 密码**
- 或 **Gumroad API Key**（更安全）

### 用来做什么
- 自动更新产品页面（标题、描述、价格）
- 查看销售数据
- A/B 测试不同价格/描述
- 自动发布新产品

### 如何提供
1. 登录 https://app.gumroad.com/settings/advanced
2. 生成 API Key
3. 把 API Key 给我

**安全建议**: Gumroad API 可以设置只读或读写权限
- 初期可以给只读（查看数据）
- 确认我正常工作后，给写入（自动更新）

---

## 3. Reddit 权限

### 需要什么
- **Reddit 账号邮箱 + 密码**
- 或 **Reddit API (App ID + App Secret + Refresh Token)**

### 用来做什么
- 自动在相关 subreddit 发帖
- 自动回复评论
- 自动监测帖子表现

### 如何提供
**方案A - 使用你的主账号**
- 提供账号密码
- 风险：如果操作不当可能影响你的账号声誉

**方案B - 创建专用推广账号（推荐）**
- 注册新 Reddit 账号（如: OpenClawTools）
- 专门用于推广我们的工具
- 养号几天再开始发推广内容

**建议**: 先用方案B，更灵活且不影响你的主账号

## 5. Hacker News（特殊说明）

### 需要什么
**不需要独立权限**

## 🔐 安全建议

### 推荐做法
1. **先用 API Key 而非密码**（更细粒度控制）
2. **从只读权限开始**（我可以查看数据，但不能修改）
3. **逐步放开权限**（确认我正常工作后再给写入）
4. **使用专用账号**（Reddit 用新账号，不影响主号）

### 你可以随时
- 撤销 API Key
- 修改密码
- 查看操作日志
- 限制我的权限
